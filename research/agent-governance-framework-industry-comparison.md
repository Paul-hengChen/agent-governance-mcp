# Agent-Governance-MCP vs. Industry-Mainstream Agent Frameworks & Constitution Conventions

> Researcher findings. Depth: **deep**. Method: `/deep-research` harness (5 search angles → 15+ fetched sources → 107 adversarially-verified claims, 104 confirmed at high confidence). Synthesised here after the workflow's final synth step was interrupted; the verified claim base was salvaged from the run journal.
> Date: 2026-05-31. Recency gate: sources older than 2024-11 tagged `(stale)`.

## Summary

- **This framework is a "workflow", not an "agent", in Anthropic's taxonomy** — LLMs orchestrated through predefined code paths (server-enforced routing chain + state-machine transitions), not LLMs dynamically directing themselves. That is a legitimate, defensible design choice, and several of its pillars (anti-loop circuit breakers, clean-context code-review, evidence-gated PASS) match Anthropic's *own* current best-practice guidance almost verbatim. [T1]
- **Its role model is mainstream and well-validated.** The pm/architect/sr-engineer/qa-engineer role-specialization-plus-handoff design maps ~1:1 onto MetaGPT's assembly-line SOP roles and is structurally similar to ChatDev's waterfall chat-chain. The *server-enforced* transition validation is actually **stronger** than what MetaGPT, ChatDev, or the OpenAI Agents SDK provide (those are advisory/model-driven). [T1]
- **The "constitution" naming is now industry-validated**, not idiosyncratic: GitHub Spec Kit ships a `.specify/memory/constitution.md` governing-principles artifact. But note the term collision — Anthropic's "Constitutional AI" is a *training-time* technique (RLAIF, shapes model weights), categorically distinct from this *runtime/operational* constitution. The framework should own that distinction explicitly. [T1]
- **The three biggest gaps vs. 2025-2026 best practice are: (1) no parallel agent execution, (2) no evaluation/observability layer, and (3) no model-routing / cost control.** Mainstream frameworks (Anthropic's own multi-agent system, OpenAI SDK tracing, LangGraph checkpointing/LangSmith, CrewAI, Google ADK) treat these as first-class. Anthropic reported a parallel orchestrator-worker system beat single-agent by **90.2%** and cut research time up to **90%**. [T1]
- **Secondary gaps:** no layered memory architecture (flat `handoff.md`/`tasks.md` vs. ADK Sessions/Memory/Artifacts), no explicit human-in-the-loop checkpoints (vs. "autonomy sliders"/approval gates), the always-injected constitution+skills bundle risks context bloat (Anthropic: keep always-on rules lean, prefer on-demand skills), and the role SOPs are MCP-prompt-only rather than aligned to the portable AGENTS.md / Agent Skills open standards. [T1][T2]

## Evidence

### Q1 — How mainstream multi-agent SE frameworks structure roles/SOPs/handoffs

- **MetaGPT** uses "an assembly line paradigm to assign diverse roles to various agents" and "encodes Standardized Operating Procedures (SOPs) into prompt sequences… allowing agents… to verify intermediate results and reduce errors", motivated explicitly by "cascading hallucinations caused by naively chaining LLMs." Its five roles (Product Manager, Architect, Project Manager, Engineer, QA) map ~1:1 onto this framework's roles; handoff is via structured document artifacts (PRD → design → tasks). — arXiv:2308.00352, ICLR 2024 Oral `(stale: 2023)` [T1]
- MetaGPT additionally uses a **global publish/subscribe message pool** so agents read shared structured messages directly rather than querying each other — an architecture this framework's file-based state parallels but does not implement as pub/sub. — MetaGPT paper / IBM Think [T1]
- **ChatDev** segments development into "three sequential phases: design, coding, and testing" following a waterfall model, with agents "guided in what to communicate (via chat chain) and how to communicate (via communicative dehallucination)." Coordination is purely conversational multi-turn dialogue between role-pairs — **no server-enforced state machine or transition validation.** (Caveat: some sources list four phases incl. documenting; "communicative dehallucination" is an anti-hallucination tactic, *not* a handoff mechanism — see Open Questions.) — arXiv:2307.07924, ACL 2024 `(stale: 2023)` [T1]
- **OpenAI Agents SDK** reduces orchestration to a few primitives — Agents, Handoffs, Guardrails (plus Sessions for memory) — favouring "few… primitives to make it quick to learn." Handoffs are "represented as tools to the LLM" (e.g. `transfer_to_refund_agent`); for multiple destinations the SDK says "register one handoff per destination and **let the model choose among them**." Routing is **model-driven**, structurally distinct from this framework's server-enforced `ALLOWED_TRANSITIONS`. — openai.github.io/openai-agents-python [T1]
- The SDK defines exactly two orchestration patterns: **Handoffs** (control transfers entirely; specialist owns the next response) and **Agents-as-tools / Manager** (manager keeps ownership of the reply, calls specialists as "bounded capabilities"). — developers.openai.com/api/docs/guides/agents/orchestration [T1]
- **LangGraph** models orchestration as "directed graphs with typed state… nodes are agents or functions, edges define transitions (including conditional routing)" — a stricter formalization than a linear chain — and persists "every state transition… enabling time-travel debugging, human-in-the-loop approvals." — practitioner analysis, 2026-05 [T3]
- **CrewAI** uses a role/goal/backstory metaphor ("crews") with built-in task delegation, sequencing, state management, and optional humans-in-the-loop — directly comparable to this framework's role-skill personas. **AutoGen/AG2** is *not* role-based but conversational (agents reach consensus through structured dialogue). — practitioner surveys, 2026-05 [T3]
- **OpenAI's explicit guidance: do not over-decompose.** "Start with one agent whenever you can. Add specialists only when they materially improve capability isolation, policy isolation, prompt clarity, or trace legibility… Splitting too early creates more prompts, more traces, and more approval surfaces without necessarily making the workflow better." — developers.openai.com [T1]

### Q2 — Emerging agent rules / "constitution" file conventions

- **AGENTS.md** is "a simple, open format for guiding coding agents… just standard Markdown" with **no required fields/schema**; "README.md files are for humans… AGENTS.md complements this by containing… build steps, tests, and conventions." Adopted by 60,000+ repos, read natively by Codex, Cursor, Copilot, Gemini CLI, Aider, Windsurf, Zed, Jules, Devin, etc., and stewarded by the Linux Foundation's Agentic AI Foundation (Dec 2025). It is **advisory, not enforced** — "explicit user chat prompts override everything." — agents.md, InfoQ, Linux Foundation [T1/T2]
- **Important nuance:** AGENTS.md is *not* a settled monopoly. Claude Code does **not** read AGENTS.md natively (uses CLAUDE.md); 2026 sources describe a "config war" of five competing formats (AGENTS.md, CLAUDE.md, .cursorrules/MDC, Copilot instructions, SKILL.md). — thepromptshelf, codersera, 2026 [T3] *(this was the one claim refuted as overreach — "de facto mainstream" is overstated; 60k adoption is real but the field is fragmented)*
- **CLAUDE.md** uses a three-layer merged hierarchy (user `~/.claude/CLAUDE.md`, project `./CLAUDE.md`, gitignored `./CLAUDE.local.md`); plain Markdown, no frontmatter, nearest-file-wins. — Anthropic docs / 2026-05 analysis [T1]
- **GitHub Spec Kit** ships a `/speckit.constitution` command that "creates or updates the `.specify/memory/constitution.md` file with… governing principles… code quality, testing standards, UX consistency, and performance." Its 7-step pipeline (constitution → specify → clarify → plan → tasks → analyze → implement) is a phased, gated workflow comparable to this framework's routing chain — though Spec Kit's steps are artifact stages driven by one agent, not distinct role personas with state handoffs. **This validates the "constitution" naming for operational governance.** — github/spec-kit, Microsoft Dev blog [T1]
- **Amazon Kiro** formalizes spec-driven development as three sequenced phases (requirements in EARS notation → design → tasks), plus **"steering files"** (`.kiro/steering/`, project- or global-scope) that hold coding standards/conventions/workflows and "guide agent responses… rather than modifying core prompts." Also offers event-triggered "agent hooks" and an "Autopilot" autonomy mode. — kiro.dev docs [T1]
- **2026 practitioner convention** separates instructions into five files by load-timing/scope: CLAUDE.md (always-on identity), AGENTS.md (always-on, agent-agnostic), MEMORY.md (persistent decision log), SKILLS.md (on-demand), CONTEXT.md (session-scoped). Recommendation: **keep always-on CLAUDE.md under 100 lines.** This framework's ~100-line constitution sits at the *upper bound* of recommended always-on size. — practitioner blog, 2026-04 [T3]

### Q3 — Anthropic's own published guidance (alignment & divergence)

- **"Building Effective Agents" (2024-12):** distinguishes **workflows** ("LLMs and tools orchestrated through predefined code paths") from **agents** ("LLMs dynamically direct their own processes"). This framework is squarely a *workflow*. Names five patterns: prompt chaining, **routing**, **parallelization** (sectioning/voting), **orchestrator-workers**, evaluator-optimizer. The framework uses routing + chaining but **not parallelization** (which Anthropic recommends "for speed or multiple perspectives"). — anthropic.com/engineering [T1]
- **ALIGNS:** Anthropic recommends "stopping conditions (such as a maximum number of iterations) to maintain control" — directly validating the **anti-loop circuit breakers** (max 2 fix attempts, hop cap 10). [T1]
- **ALIGNS:** Anthropic recommends a fresh-context adversarial reviewer that "sees only the diff and the criteria… not the reasoning that produced the change" — directly matching the **code-reviewer clean-context independence** design. [T1]
- **ALIGNS:** "Have Claude show evidence rather than asserting success: the test output… or a screenshot" — matches the **evidence-gated PASS** and **visual-fidelity loop**. [T1]
- **ALIGNS:** "Use hooks for actions that must happen every time with zero exceptions… Unlike CLAUDE.md instructions which are advisory, hooks are deterministic." — validates the choice to **server-enforce** the pre-flight/state protocol rather than rely on prose. [T1]
- **DIVERGES (context budget):** "CLAUDE.md is loaded every session, so only include things that apply broadly. For domain knowledge or workflows that are only relevant sometimes, use skills… loaded on demand without bloating every conversation." Also: "Bloated CLAUDE.md files cause Claude to ignore your actual instructions!" The framework injects constitution + role skills together — measure against this. [T1]
- **DIVERGES (parallelism + model-routing):** Anthropic's production multi-agent research system uses an **orchestrator-worker** topology — a lead (Opus) spawns 3-5 subagents (Sonnet) **in parallel**. It "outperformed single-agent Claude Opus 4 by 90.2%" and "cut research time by up to 90%." It scales effort to complexity (1 agent for simple facts, 10+ for complex research). The framework's strictly sequential chain exploits none of this. — anthropic.com/engineering, 2025-06 [T1]
- **DIVERGES (evaluation/observability):** Anthropic uses "LLM-as-judge" rubric scoring (factual accuracy, citation accuracy, completeness, source quality, tool efficiency) + "full production tracing… to diagnose why agents failed." The framework enforces *process* via transitions but has **no judged-quality or tracing dimension.** — anthropic.com/engineering [T1]
- **Context engineering (2025-09):** the discipline of "curating and maintaining the optimal set of tokens." Warns of "context rot" (recall degrades as tokens grow). Prescribes **compaction**, **structured note-taking to external memory**, and **sub-agent architectures** that return condensed 1,000-2,000-token summaries. Sub-agents (Claude Code) run in "fresh, isolated context window[s]" and support per-agent tool allowlists + model selection "to control costs by routing… to faster, cheaper models like Haiku." — anthropic.com/engineering, Claude Code docs [T1]
- **Writing tools for agents (2025-09):** "More tools don't always lead to better outcomes"; namespace tools with shared prefixes (validates the `tw_*` convention); token-budget responses (Claude Code caps tool responses at 25,000 tokens); and **evaluation-driven tool iteration** (build evals, analyze transcripts, refine) — an explicit eval layer the framework lacks. — anthropic.com/engineering [T1]
- **Agent Skills (2025-10):** open standard (agentskills.io) with **progressive disclosure** (metadata → SKILL.md body → linked files on demand) so agents "don't… read the entirety of a skill into context." Keep SKILL.md < 500 lines. `context: fork` runs a skill in an isolated subagent; `disable-model-invocation`/`user-invocable` give built-in human-in-the-loop gating for side-effecting workflows. — anthropic.com/engineering [T1]

### Q4 — "Constitutional AI" vs. operational constitution (the term collision)

- **Constitutional AI (Bai et al., arXiv:2212.08073) is a TRAINING-TIME technique**, not a runtime ruleset: it trains "a harmless AI assistant through self-improvement, without any human labels identifying harmful outputs," via two phases — supervised self-critique/revision, then **RL from AI Feedback (RLAIF)** — shaping **model weights**. The "constitution" is consumed during fine-tuning; "once deployed, Claude exhibits these trained behaviors implicitly rather than consulting principles at runtime." `(stale: 2022, but canonical)` — Anthropic [T1]
- **Anthropic's "Constitution" for Claude (2026-01 update)** is "the foundational document that both expresses and shapes who Claude is" — a values/ethics/safety document (Helpfulness, Guidelines, Ethics, Safety, Nature), used to construct synthetic training data. It deliberately "teach[es] the model to reason about *why*… not just *what* to do." It is a **training and steering guide whose primary audience is the model itself**, explicitly *not* a per-project operational protocol enforced by tooling. — Anthropic, 2026-01 [T1]
- **The distinction is now drawn in the literature:** "runtime constitutions enforce governance constraints during agent execution, not just during training… while Constitutional AI shapes model behavior through RLHF during development, runtime constitutions operate as deterministic operational checks during deployment." Training-time alignment "degrades when agents operate in open-ended environments," motivating verifiable runtime enforcement. — practitioner sources, 2026-02 [T3]
- **Implication:** this framework's "Constitution" is correctly an *operational/runtime* constitution (the genuinely useful, complementary layer CAI cannot provide — CAI "cannot prevent duplicate actions, hallucinated references, or unauthorized operations"). The naming is defensible but should be disambiguated to avoid conflation. [T3]

### Q5 — Prioritized improvement opportunities (with industry precedent)

- **Parallel execution** — Anthropic orchestrator-worker (+90.2% / -90% time); LangGraph parallel nodes; Claude Code subagents/`context: fork`; OpenAI agents-as-tools. The sequential chain leaves this entirely on the table. [T1]
- **Evaluation/observability layer** — LLM-as-judge rubric + production tracing (Anthropic); OpenAI built-in tracing; LangSmith trace-level visibility; Google ADK strongly-typed Event records; OpenTelemetry GenAI semantic conventions (traces/metrics/logs) being standardized so telemetry is portable and feeds evals. The framework has *drift detection* but no judged quality or tracing. [T1][T2]
- **Model-routing / cost control** — heterogeneous models per role (Opus lead + Sonnet/Haiku workers); Claude Code per-subagent model selection + tool allowlists "to control costs." The framework has no model-tier assignment per role. [T1]
- **Layered memory** — Google ADK separates Sessions (durable logs) / Memory (searchable long-term) / Artifacts (large data), with reactive vs. proactive recall; production memory systems (mem0) converged on async writes, reranking, metadata filtering, timestamps, multi-scope (`user/agent/run/org`) — saving ~6,900 vs ~26,000 tokens/conversation. The flat `handoff.md`/`tasks.md` is a single state file. [T1][T2]
- **Explicit human-in-the-loop checkpoints** — LangGraph HITL approvals; Kiro Autopilot; "autonomy sliders" with dollar/approval limits (Ramp: $50 conservative / $500 aggressive); Claude Code `disable-model-invocation` gating. Anthropic: "human review remains crucial… automated testing alone is insufficient." The framework's gates are role-to-role, not human-approval parameterized. [T1][T3]
- **Context-budget discipline** — split always-on rules (lean, broadly-applicable) from on-demand skills (progressive disclosure, < 500 lines). Audit whether the full constitution+skills bundle must be injected every session. [T1]
- **Open-standard alignment** — role SOPs live only in MCP prompts; aligning them to the portable Agent Skills / AGENTS.md conventions would improve cross-tool reuse (with the caveat that the standards field is still fragmented). [T1][T3]

## Recommendation

**Keep the core architecture — it is sound and, on enforcement, ahead of the field — and invest the next cycle in observability + selective parallelism, in that order.**

Rationale (cost/risk/effort):
1. **Add an evaluation/observability layer first (highest value, moderate effort, low risk).** Emit structured trace events per `tw_*` transition (role, round, tokens, duration, outcome) and add an optional LLM-as-judge rubric on QA PASS (factual/citation/completeness/coverage). This is the single biggest gap vs. *every* mainstream framework, it is additive (no architecture change), and it produces the data needed to justify every later change. Align event shape to OpenTelemetry GenAI conventions for portability.
2. **Add model-routing per role (high value, low effort, low risk).** Declare a model tier per role skill (e.g. researcher/code-reviewer = Opus; sr-engineer = Sonnet; doc-writer/status = Haiku). Anthropic's own data shows heterogeneous routing is both cheaper and higher-quality.
3. **Introduce bounded parallelism where the chain is already independent (high value, higher effort, medium risk).** The `sr-engineer ↔ code-reviewer` loop and multi-task batches are the natural candidates; fan-out QA across independent task ids, or run researcher sub-questions in parallel (this very report is the proof case). Preserve the server-enforced transition invariants — parallelism inside a phase, not across the gated chain.
4. **Disambiguate the "constitution" term** in the constitution header and docs: one sentence stating this is a *runtime/operational* constitution, explicitly distinct from Anthropic's *training-time* Constitutional AI. Cheap, removes a real conceptual confusion, and is now backed by Spec Kit's identical naming.

Defer (lower urgency): layered memory architecture and autonomy-slider HITL — valuable but larger, and best designed *after* the observability layer reveals where they pay off.

## Alternatives Considered

- **Adopt model-driven handoffs (OpenAI SDK style) instead of server-enforced transitions.** *Rejected.* The server-enforced `ALLOWED_TRANSITIONS` + evidence gates are this framework's chief differentiator and align with Anthropic's "use deterministic hooks for things that must happen with zero exceptions." Model-driven routing trades that guarantee for flexibility the cross-IDE governance use-case does not want.
- **Migrate role SOPs wholesale to AGENTS.md / Agent Skills open standards.** *Rejected (for now).* Real portability upside, but (a) the standard field is fragmented ("config war"; Claude Code doesn't read AGENTS.md), and (b) those formats are *advisory* — they cannot express the server-enforced state machine that is the point of this server. Better as an *export/interop* layer than a replacement.
- **Go fully autonomous / "agent" (let the LLM direct the chain).** *Rejected.* Contradicts Anthropic's workflow-vs-agent guidance for predictable, governable tasks; would forfeit determinism, auditability, and cross-session reproducibility.
- **Do nothing / status quo.** *Rejected.* The architecture is competitive, but the *absence* of any evaluation/observability layer is the one gap shared by no mainstream peer — it blocks data-driven improvement and is the lowest-risk thing to fix.
