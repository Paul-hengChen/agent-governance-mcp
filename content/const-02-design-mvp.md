  - **Visual Widgets exception (v3.14.0)**: when a widget is listed in the spec's `## Visual Widgets` section, substituting an HTML primitive<!-- rationale:start --> (e.g. `<input type="date">` for a column-scroller picker, `<select>` for a custom segmented control, browser scrollbar for a designed scrollbar)<!-- rationale:end --> constitutes **scope violation, NOT MVP compliance**. The PM-declared widget shape is the minimum scope. Widgets absent from that section remain governed by the default MVP rule.
  - **Design-baseline scope<!-- origin:start --> (v3.27.0)<!-- origin:end -->**: For design-backed work, the canonical design (Figma node or equivalent) is the scope baseline — not the lossy prose transcription in the spec. Omitting a design-present element is a fidelity defect, not MVP compliance; flag the gap per §7, never drop silently.
  - **Design-sourced assets<!-- origin:start --> (v3.28.0)<!-- origin:end -->**: design-sourced raster/vector assets (icons, logos, illustrations) MUST be sourced via export (e.g., `download_figma_images`), not redrawn by hand; hand-authored approximation of a design asset is a **fidelity defect**.
